# 🔒 安全与隐私声明 (Security & Privacy Policy)

**项目名称**：安眠匣 (SleepHaven)  
**版本**：v1.0.5
**最后更新**：2026-03-17

---

## 1. 核心原则

安眠匣 (SleepHaven) 将用户的隐私安全和数据主权置于首位。我们遵循 **“最小化收集”** 和 **“本地优先”** 的设计原则。本应用不建立用户账户系统，不收集个人身份信息 (PII)，所有核心数据均存储在用户本地设备中。

---

## 2. 数据收集与存储

### 2.1 本地存储 (Local Storage)
本应用产生的所有敏感数据均仅存储在用户设备的本地存储 (`uni.setStorageSync` / `LocalStorage`) 中。
- **存储内容**：
  - 睡眠记录（开始时间、结束时间、时长、唤醒次数）。
  - 睡眠负债累计值。
  - 用户偏好设置（主题模式、有效睡眠阈值）。
  - AI 生成的建议文本。
- **数据归属**：数据完全归属于用户，开发者无法远程访问、查看或备份这些数据。
- **数据清除**：用户可随时在应用内删除单条记录或重置所有数据，卸载应用亦会自动清除所有本地数据。

### 2.2 网络传输 (Network Transmission)
本应用仅在请求 **AI 睡眠建议** 功能时发生网络通信。
- **传输内容**：仅发送经过脱敏的**统计数字**至 AI 服务商。
  - *示例 payload*：`"我刚刚完成了夜间睡眠，时长 420 分钟，评分 85/100，中途亮屏 1 次。请给出建议。"`
- **不包含的信息**：
  - ❌ 不包含用户姓名、手机号、地理位置、设备 IMEI 等个人身份信息。
  - ❌ 不包含具体的睡眠开始/结束时间戳（仅计算后的时长）。
  - ❌ 不包含录音、摄像头画面或其他传感器原始数据。
- **传输加密**：所有 API 请求均通过 **HTTPS (TLS 1.2+)** 加密传输，防止中间人攻击。

---

## 3. 第三方服务与安全

### 3.1 AI 服务提供商
本应用使用 **SiliconFlow (硅基流动)** 作为大模型推理服务提供商。
- **API 密钥管理**：
  - 当前版本（开源/演示版）中，API Key 硬编码在客户端代码中。**⚠️ 风险提示**：在公开部署的 H5 环境中，恶意用户可能通过查看源代码获取此 Key。
  - **生产环境建议**：正式商用或公开发布时，**务必**将 API 请求移至云函数 (Cloud Functions) 或后端服务器代理，严禁将私钥暴露在客户端代码中。
- **数据隐私**：SiliconFlow 的数据处理遵循其官方隐私政策。我们仅发送必要的上下文文本以生成建议，不用于模型训练（取决于具体服务商策略，建议用户知悉）。

### 3.2 外部资源加载
- **Markdown 解析库**：应用通过 CDN (`cdn.jsdelivr.net`) 动态加载 `marked.js` 库。
  - **完整性校验**：建议在后续版本中引入 SRI (Subresource Integrity) 校验，或直接将库文件打包至本地，防止 CDN 被篡改导致 XSS 攻击。

## 4. 权限使用说明

本应用遵循最小权限原则，仅申请实现功能所必需的系统权限：

| 权限 | 用途 | 是否强制 |
| :--- | :--- | :--- |
| **本地存储** | 保存睡眠记录和设置 | 是 (核心功能) |
| **网络请求** | 获取 AI 建议 | 否 (仅在使用 AI 功能时需要) |
| **前台/后台事件监听** | 检测用户是否中途亮屏/切出应用 | 是 (用于计算睡眠连续性) |
| **振动反馈** | 检测到中途唤醒时提供轻微触觉反馈 | 否 (可关闭) |

**❌ 本应用绝不申请以下权限**：
- 麦克风录音
- 摄像头拍摄
- 精确地理位置
- 通讯录/短信/通话记录
- 设备唯一标识符 (IMEI/OAID)

## 5. 潜在风险与缓解措施

### 5.1 客户端 API Key 泄露
- **风险**：当前版本 Key 写在前端，可能被提取并盗用额度。
- **缓解**：
  1.  在 SiliconFlow 控制台设置 **Referer 白名单** 或 **IP 白名单**（如果支持）。
  2.  设置每日/每月 **消费限额**。
  3.  **长期方案**：重构架构，引入轻量级云函数作为中间层代理 API 请求。

### 5.2 本地数据丢失
- **风险**：用户清除浏览器缓存、卸载 App 或更换设备会导致数据永久丢失。
- **缓解**：
  - 在设置中明确提示用户“数据仅存于本地”。
  - 未来版本可考虑增加“导出 JSON 备份”功能，让用户自行保存数据。

### 5.3 依赖库供应链攻击
- **风险**：引用的 CDN 资源若被篡改，可能执行恶意脚本。
- **缓解**：
  - 优先将 `marked.js` 等第三方库下载并打包至项目本地 (`static` 目录)，减少对外部网络的依赖。
